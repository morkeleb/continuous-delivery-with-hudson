Hudson pode gravar o 'fingerprint' dos arquivos (mais frequentemente do arquivos jar) para manter registro de onde/quando estes arquivos são produzidos e usados. Quando você tiver projetos inter-dependêntes no Hudson, isto permite que você rapidamente encontre respostas para questões como:
  • Eu tenho um foo.jar no meu HD mas qual o número da construção de FOO que gerou ele?
  • Meu projeto BAR depende de foo.jar do projeto FOO.
    • Qual construção de foo.jar é usada na construção de BAR número 51?
    • Qual construção de BAR contém minha correção de bug para o foo.jar de número 32?

Para usar esta característica, todos os projetos envolvidos (não apenas o projeto no qual um arquivo é produzido, mas também os projetos nos quais o arquivo é usado) necessitam usar isto e gravar os fingerprints.
