Demande à Hudson de scruter les changements dans l'outil de gestion de version.

Notez que cette opération est consommatrice de ressources pour CVS, car chaque scrutation signifie que Hudson va scanner l'ensemble du workspace et le comparer avec le serveur. Envisagez d'utiliser un trigger de type 'push' pour éviter cette surcharge, comme décrit dans ce document.
